const a = Number(document.getElementById("a").value); 
const b = Number(document.getElementById("b").value); 
const op = document.getElementById("op").value; 
let res = 0; 
if (op === "+") res = a + b; 
if (op === "-") res = a - b; 
if (op === "*") res = a * b; 
if (op === "/") res = b !== 0 ? a / b : "Error"; 
document.getElementById("result").textContent = res; 
}

1) Git branching story — each branch adds a chapter → merge into final book.html
Goal: create branches chapter-1, chapter-2, chapter-3 (for example). Each branch adds a chapter file. Merge all chapters into main as book.html.
Files / example content
•	chapter1.html
<h2>Chapter 1 — The Beginning</h2>
<p>Once upon a time...</p>
•	chapter2.html
<h2>Chapter 2 — The Journey</h2>
<p>The hero set out on a long journey...</p>
•	chapter3.html
<h2>Chapter 3 — The End</h2>
<p>They returned home, wiser and kinder.</p>
Steps (local Git)
1.	Create repo (if you already have a repo skip to step 2).
mkdir branching-story
cd branching-story
git init
2.	Create a README.md and initial commit (optional but recommended).
echo "# Branching Story" > README.md
git add README.md
git commit -m "Initial commit"
3.	Create branch chapter-1 and add chapter1.
git checkout -b chapter-1
cat > chapter1.html <<'EOF'
<h2>Chapter 1 — The Beginning</h2>
<p>Once upon a time...</p>
EOF
git add chapter1.html
git commit -m "Add chapter 1"
4.	Create chapter-2 from main and add chapter2.
git checkout main        # or git switch main (if git >= 2.23)
git checkout -b chapter-2
cat > chapter2.html <<'EOF'
<h2>Chapter 2 — The Journey</h2>
<p>The hero set out on a long journey...</p>
EOF
git add chapter2.html
git commit -m "Add chapter 2"
5.	Create chapter-3.
git checkout main
git checkout -b chapter-3
cat > chapter3.html <<'EOF'
<h2>Chapter 3 — The End</h2>
<p>They returned home, wiser and kinder.</p>
EOF
git add chapter3.html
git commit -m "Add chapter 3"
6.	Merge branches into main (option A: fast/clean local merge)
git checkout main
git merge --no-ff chapter-1 -m "Merge chapter-1"
git merge --no-ff chapter-2 -m "Merge chapter-2"
git merge --no-ff chapter-3 -m "Merge chapter-3"
--no-ff keeps each chapter as an obvious merge commit in history (optional).
7.	Create the final book.html that assembles the chapters. Simple approach: concatenate the files.
cat chapter1.html chapter2.html chapter3.html > book.html
git add book.html
git commit -m "Create combined book.html from chapters"
8.	Push to GitHub (replace with your origin):
git remote add origin https://github.com/<your-username>/<repo>.git
git branch -M main
git push -u origin main
# push branches (if you want to keep them remote)
git push origin chapter-1 chapter-2 chapter-3
Notes / Alternatives
•	PR-based workflow: push each branch to GitHub, open a Pull Request for each chapter, then merge on GitHub. This is good if you want reviewers.
•	If merges cause conflicts: Git will show conflicts. Open the conflicting file, manually edit to resolve, then:
git add <file>
git commit
•	Order control: concatenation order determines the chapter order. If chapters evolve later, update book.html accordingly.

3) Multi-stage Git branching task — each branch stores part of a secret message → merge into main
Goal: branches part-1, part-2, part-3 each contain a file with a fragment. After merging all into main, combine fragments into secret_message.txt.
Example fragments
•	part1.txt → THE
•	part2.txt → QUICK
•	part3.txt → BROWN FOX
Steps (local)
1.	Initialize repo (if needed):
mkdir secret-branches
cd secret-branches
git init
echo "# Secret branches" > README.md
git add README.md
git commit -m "Initial commit"
2.	Create part-1 branch and file:
git checkout -b part-1
echo "THE " > part1.txt
git add part1.txt
git commit -m "Add part1 fragment"
git push -u origin part-1
3.	Create part-2 branch:
git checkout main
git checkout -b part-2
echo "QUICK " > part2.txt
git add part2.txt
git commit -m "Add part2 fragment"
git push -u origin part-2
4.	Create part-3 branch:
git checkout main
git checkout -b part-3
echo "BROWN FOX" > part3.txt
git add part3.txt
git commit -m "Add part3 fragment"
git push -u origin part-3
5.	Merge parts into main:
git checkout main
git merge --no-ff part-1 -m "Merge part-1"
git merge --no-ff part-2 -m "Merge part-2"
git merge --no-ff part-3 -m "Merge part-3"
6.	Create the final secret_message.txt by concatenating in the correct order:
# simple concat (ensure order is part1, part2, part3)
cat part1.txt part2.txt part3.txt > secret_message.txt
# trim extra spaces if you want in one line
tr -d '\n' < secret_message.txt > tmp && mv tmp secret_message.txt
# or manual edit secret_message.txt to create final
git add secret_message.txt
git commit -m "Assemble final secret_message.txt"
git push origin main
Alternative: merge strategy where the final file is built during merging
If you want the main merge to automatically contain the assembled message, you can:
•	Have each branch add a fragment.txt in a fragments/ folder.
•	After merging, run a small script to produce secret_message.txt (this is the same concatenation step but done as a post-merge step).
Handling conflicts
If two branches modify the same file (e.g., secret.txt) differently, you will get a merge conflict. Resolve manually:
# after git merge shows conflict
# open file, resolve, then:
git add resolved-file
git commit
